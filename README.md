# AegisTalk Public Web E2EE

Публичная копия проекта только для веб-версии.

## Что внутри

- `frontend/` - React/Vite веб-клиент без мобильных зависимостей.
- `backend/` - Node.js API и WebSocket-сервер для веб-клиента.
- Мобильная часть, приватные ключи, боевые fallback-адреса и лишние backup-файлы удалены.

## E2EE

- Текстовые сообщения и JSON-метаданные вложений шифруются на клиенте.
- Сервер хранит и отдаёт только зашифрованный payload.
- Push-превью в публичной версии не раскрывают текст сообщения.
- Для группового чата клиент использует публичные ключи участников.

## Важные ограничения

- Серверный AI из публичной копии удалён: он несовместим с честным end-to-end шифрованием без отдельного согласия пользователя.
- Приватные ключи хранятся локально в браузере и автоматически между устройствами не переносятся.

## Локальный запуск

1. Скопируйте `frontend/.env.example` в `frontend/.env`.
2. Скопируйте `backend/.env.example` в `backend/.env`.
3. Поднимите локальную базу:

```bash
docker compose up -d postgres
```

4. Установите зависимости:

```bash
cd frontend && npm install
cd ../backend && npm install
```

5. Запустите backend:

```bash
cd backend
npm run dev
```

6. Запустите frontend:

```bash
cd frontend
npm run dev
```

## Что делает backend при первом старте

- Подключается к локальному Postgres без принудительного SSL.
- Если база пустая, сам накатывает базовую схему из `backend/db/init.sql`.
- Затем применяет недостающие миграции из `backend/db/migrations`.

После этого публичная веб-копия должна работать локально так же, как текущая рабочая версия, если `.env` заполнен и Postgres доступен на `localhost:5432`.
